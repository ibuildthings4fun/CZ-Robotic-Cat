# 架构说明

CZ Robotic Cat 系统大致分为两部分：

1. **后端服务（backend）**
   - 使用 Python 编写；
   - 定期从交易所 / API 获取 BNB 的 24 小时行情数据；
   - 调用行为引擎，根据行情计算出一个高层状态（CALM / ALERT / CAUTIOUS / PANIC）；
   - 将状态以 JSON 形式发布到 MQTT 服务器上的 `cat/state` 主题。

2. **嵌入式固件（firmware）**
   - 运行在 ESP32 上，使用 Arduino 框架开发；
   - 通过 Wi‑Fi 连接到同一台 MQTT 服务器；
   - 订阅 `cat/state` 主题，解析收到的状态；
   - 根据状态控制机器人猫的眼睛颜色、头部和尾巴舵机角度等。

整体的设计目标不是做一个「聪明的交易机器人」，而是做一个能用实体动作可视化 BNB 市场情绪的小玩具，同时把「CZ 风格」抽象成一种冷静、偏安全、偏长期的行为逻辑。
